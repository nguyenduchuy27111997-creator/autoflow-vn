# Codebase Structure

## Directory Purposes

**src/app/:**
- Purpose: Next.js App Router with file-based routing
- Contains: Page components, API routes, layouts
- Key structure:
  - `layout.tsx` - Root layout with global metadata and scripts
  - `page.tsx` - Homepage marketing site
  - `api/` - Backend endpoints (audit, PDF delivery)
  - Marketing pages: `bang-gia/`, `dich-vu/`, `tai-lieu/`, `blog/`, `ket-qua/`, `dieu-khoan/`, `chinh-sach-bao-mat/`
  - Protected area: `portal/` (login + dashboard)
  - Content pages: `audit/`, `quiz/`, `ve-toi/`, `tu-lam/`
  - SEO files: `robots.ts`, `sitemap.ts`

**src/components/:**
- Purpose: Reusable React components (mostly "use client")
- Contains: Marketing sections (Hero, Navbar, Pricing, FAQ), UI primitives, form helpers
- Organization: Top-level components (Hero, Footer, Integrations) + `ui/` subdirectory for primitives
- Key files:
  - `Navbar.tsx` - Navigation with scroll effects and dropdown menus
  - `Hero.tsx` - Main value proposition with video and counter animation
  - `Footer.tsx` - Global footer with links and CTA
  - `ui/Button.tsx` - Reusable button with variant/size props
  - `ui/Container.tsx` - Max-width layout wrapper
  - `ui/FormField.tsx` - Form input wrapper
  - `JsonLd.tsx` - Schema markup injector

**src/lib/:**
- Purpose: Utility functions and client initialization
- Contains: Supabase client setup, middleware helpers
- `supabase/`:
  - `server.ts` - Server-side Supabase client with cookie handling
  - `client.ts` - Browser-side Supabase client
  - `middleware.ts` - Session refresh and auth protection

**src/data/:**
- Purpose: Static configuration and content data
- Contains: Form options, blog metadata, quiz questions
- Key files:
  - `constants.ts` - Industries, team sizes
  - `blog-posts.ts` - Blog metadata for listing/SEO
  - `quiz-data.ts` - Quiz questions and answers

**public/:**
- Purpose: Static assets served directly
- `videos/` - Hero video, demo videos
- `pdfs/` - Lead magnet PDFs (AutoFlow-Lead-Magnet-10-Quy-Trinh.pdf)

**.planning/codebase/:**
- Purpose: Architecture and analysis documents (ARCHITECTURE.md, STRUCTURE.md, etc.)
- Note: Generated by GSD tooling, not committed to main repo

## Naming Conventions

**Files:**
- PascalCase: React components (`Hero.tsx`, `Button.tsx`, `Navbar.tsx`)
- kebab-case: API routes (`audit/route.ts`, `tai-lieu/route.ts`)
- camelCase: Utility functions (`createClient.ts`, `updateSession.ts`)
- lowercase: Config files (`next.config.ts`, `middleware.ts`)

**Directories:**
- kebab-case: App router directories (`/dich-vu/`, `/tai-lieu/`, `/portal/dashboard/`)
- lowercase: Feature directories (`lib/`, `data/`, `components/`, `public/`)

**Components:**
- PascalCase filename matches export name (`Button.tsx` → `export default function Button()`)
- UI primitives grouped in `components/ui/` subdirectory
- Business/marketing components in `components/` root

**Variables & Functions:**
- camelCase: Variables, functions, hooks (`userEmail`, `handleSubmit`, `isRateLimited`)
- UPPER_SNAKE_CASE: Constants (`MAX_REQUESTS`, `RATE_LIMIT_WINDOW`, `GA_ID`)
- Prefixed with `is`, `has`, `can`: Booleans (`scrolled`, `submitted`, `loading`)

## Where to Add New Code

**New Marketing Page:**
- Create directory: `src/app/[slug]/`
- Add file: `src/app/[slug]/page.tsx` (export page component)
- Add layout if needed: `src/app/[slug]/layout.tsx`
- Use `Container` + `Navbar` + `Footer` components
- Set metadata in page export or layout
- Route-based slug automatically matches directory name

**New API Endpoint:**
- Create directory: `src/app/api/[feature]/`
- Add file: `src/app/api/[feature]/route.ts` (export POST/GET/etc functions)
- Import rate limiting logic if accepting user input
- Use `createClient()` from `src/lib/supabase/server.ts` for database access
- Return `NextResponse.json()` with appropriate status codes

**New Component:**
- Reusable/primitive: Create in `src/components/ui/ComponentName.tsx`
- Feature-specific: Create in `src/components/ComponentName.tsx`
- Client-only component: Add `"use client"` directive at top
- Accept props with TypeScript interfaces
- Use Tailwind classes for styling

**New Form Section:**
- Create component in `src/components/FormName.tsx`
- Add `"use client"` directive (forms need interactivity)
- Import `FormField`, `Button` from `components/ui/`
- Use `useState` for form state
- POST to appropriate `/api/route` endpoint
- Handle loading/error states with user feedback

**Utilities/Helpers:**
- Shared helpers: `src/lib/[feature]/helper.ts`
- Data selectors: `src/data/[domain].ts`
- Keep Supabase client separation: `src/lib/supabase/` for auth only

**Protected Pages:**
- Create in `src/app/[route]/page.tsx`
- Add server-side auth check: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser();`
- Redirect if not authenticated: `if (!user) redirect("/portal");`
- Middleware.ts automatically protects `/portal/dashboard/*` routes

## Special Directories

**public/:**
- Purpose: Static files served at root URL
- Generated: No
- Committed: Yes (except PDFs tracked separately)
- Images/videos: Directly referenced in components (`/videos/hero.webm`)
- PDFs: Read server-side in API routes, not directly accessible to clients

**.next/:**
- Purpose: Next.js build output
- Generated: Yes (by `npm run build`)
- Committed: No (ignored in .gitignore)
- Contains: Compiled pages, server components, static exports

**node_modules/:**
- Purpose: Installed dependencies
- Generated: Yes (by npm install)
- Committed: No (ignored in .gitignore)
- Managed by: package-lock.json

**.planning/codebase/:**
- Purpose: GSD architecture analysis documents
- Generated: Yes (by GSD mapping tools)
- Committed: Yes (planning documents are version controlled)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, etc.

## Import Paths

**Path Alias:**
- `@/*` maps to `src/*` (configured in `tsconfig.json`)
- Usage: `import Button from "@/components/ui/Button"`
- Never use relative imports; always use `@/` for clarity

**Import Order (observed pattern):**
1. Next.js imports (`next/..., next/navigation, next/server`)
2. React imports (`react`, `react-dom`)
3. Third-party imports (`@supabase/...`, resend, etc.)
4. Local imports (`@/components`, `@/lib`, `@/data`)
5. Styling (Tailwind classes inline or separate CSS)
